3. KOMMUNIKATIONSHISTORIE
   ----------------------------------------------------------------------------
   KRITISCHER BEFUND: Es wurden 112.110 historische NetFlow-Einträge 
   zwischen diesen beiden IP-Adressen gefunden!
   
   Diese extrem hohe Anzahl von Verbindungen zeigt ein systematisches 
   Angriffsmuster. Alle historischen Flows weisen identische Merkmale auf:
   - Identische TCP_FLAGS (219/27)
   - Identischer Zielport (80 - HTTP)
   - Identisches L7_PROTO (7.0 - HTTP)
   - Wechselnde Quellports (typisch für automatisierte Angriffe)
   - Nahezu identische Bytegrößen und Paketmengen

4. PROTOKOLLANALYSE
   ----------------------------------------------------------------------------
   - Layer 4: TCP (Protokoll-ID 6)
   - Layer 7: HTTP (Protokoll-ID 7)
   - Zielport: 80 (Standard HTTP)
   
   Die Kombination deutet auf einen HTTP-basierten Angriff hin.

5. TCP-FLAGS ANALYSE (KRITISCH)
   ----------------------------------------------------------------------------
   - TCP_FLAGS: 219 (Dezimal) = 11011011 (Binär)
   - CLIENT_TCP_FLAGS: 219 = SYN, FIN, RST, PSH, ACK, URG
   - SERVER_TCP_FLAGS: 27 = SYN, FIN, PSH, ACK
   
   ANOMALIE ERKANNT: Die ungewöhnliche Kombination mehrerer TCP-Flags 
   (besonders RST und URG gleichzeitig mit anderen Flags) ist ein starker 
   Indikator für automatisierte Angriffswerkzeuge wie HOIC oder Brute-Force-
   Tools.

6. TRAFFIC-CHARAKTERISTIKEN
   ----------------------------------------------------------------------------

   - OUT_BYTES: 1.147 (Antworten vom Server)
   - IN_PKTS: 5
   - OUT_PKTS: 5
   - NUM_PKTS_UP_TO_128_BYTES: 8 (hoher Anteil kleiner Pakete)
   - NUM_PKTS_256_TO_512_BYTES: 1
   - NUM_PKTS_512_TO_1024_BYTES: 1
   
   INTERPRETATION: Das Muster kleiner, schneller Anfragen mit größeren 
   Serverantworten ist typisch für Web Brute-Force-Angriffe, bei denen 
   Login-Versuche oder Parameter-Fuzzing durchgeführt werden.

7. TTL-ANALYSE
   ----------------------------------------------------------------------------
   - MIN_TTL: 127
   - MAX_TTL: 127
   
   Ein konstanter TTL-Wert von 127 (typischer Windows-Startwert minus 1 Hop) 
   deutet auf ein automatisiertes Tool hin, das von einem einzelnen System 
   aus operiert.

                      EMPFOHLENE GEGENMASSNAHMEN

SOFORTMASSNAHMEN (Priorität: KRITISCH):

1. IP-BLOCKIERUNG
   → Die Quell-IP 18.218.115.60 sollte umgehend auf der Firewall und 
     im WAF (Web Application Firewall) blockiert werden.
   → Implementierung einer Blockliste für das gesamte AS16509 
     (Amazon AWS) prüfen, falls legitimer Traffic nicht betroffen ist.

2. RATE LIMITING
   → Implementierung aggressiver Rate-Limiting-Regeln für Port 80/443
   → Maximale Anfragen pro IP: 100/Minute empfohlen

3. WEB APPLICATION FIREWALL (WAF)
   → Aktivierung von Brute-Force-Schutzregeln
   → SQL-Injection- und XSS-Filter aktivieren
   → CAPTCHA für Login-Seiten implementieren

4. LOGGING & MONITORING
   → Erweiterte Protokollierung für den Zielserver aktivieren
   → Alarmierung bei wiederholten Zugriffsversuchen einrichten
   → HTTP-Anfragen auf verdächtige Payloads analysieren

LANGFRISTIGE MASSNAHMEN:

5. INFRASTRUKTUR
   → Prüfung, ob der Webserver (172.31.69.28) öffentlich erreichbar 
     sein muss oder hinter einem Reverse-Proxy platziert werden kann
   → Implementierung eines Content Delivery Networks (CDN) mit 
     DDoS-Schutz

6. AUTHENTIFIZIERUNG
   → Multi-Faktor-Authentifizierung (MFA) für alle Webanwendungen
   → Account-Lockout nach fehlgeschlagenen Anmeldeversuchen
   → Überprüfung auf kompromittierte Zugangsdaten

• Die Quell-IP 18.218.115.60 (AWS, Ohio) ist eine bekannte Angriffsquelle 
  mit dokumentierter Historie für DDOS, Brute-Force und Web-Application-
  Angriffe.

• Mit über 112.000 historischen Verbindungen zwischen denselben Endpunkten 
  liegt ein systematischer, automatisierter Angriff vor.

• Die Charakteristiken des Traffics (TCP-Flags, Paketgrößen, Portnuztung) 
  entsprechen den Signaturen eines Web Brute-Force-Angriffs.

• Das Ziel ist ein interner Webserver, der aktiv angegriffen wird und 
  Schutzmaßnahmen erfordert.

HANDLUNGSEMPFEHLUNG:
Sofortige Implementierung der empfohlenen Gegenmaßnahmen, insbesondere 
die Blockierung der Quell-IP und Verstärkung der WAF-Regeln.
